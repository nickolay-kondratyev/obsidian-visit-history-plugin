## Approach

Two options:

### Option A: Separate CSS file (recommended)

Create `styles.css` at plugin root (Obsidian convention). Scope everything:

```css
/* vault heatmap — scoped styles */
.vault-heatmap-view {
  /* Map prototype vars → Obsidian theme vars */
  --vt-bg:       var(--background-primary);
  --vt-surface:  var(--background-secondary);
  --vt-border:   var(--background-modifier-border);
  --vt-border2:  var(--background-modifier-border-hover);
  --vt-text:     var(--text-normal);
  --vt-text-dim: var(--text-muted);
  --vt-text-mut: var(--text-faint);

  /* File-type colors (no Obsidian equivalent) */
  --vt-md:       #4a5ed4;
  --vt-md-h:     #6070e8;
  --vt-md-b:     #3248b0;
  --vt-md-fg:    #a0b0ff;
  --vt-canvas:   #be7220;
  --vt-cn-h:     #d48832;
  --vt-cn-b:     #9a5a10;
  --vt-cn-fg:    #f0c070;
  --vt-exd:      #1e9e8e;
  --vt-ex-h:     #2cb4a2;

/* Then port all prototype styles with .vault-heatmap-view prefix */
/* and replace --bg → --vt-bg, --text → --vt-text, etc. */
```

### Option B: CSS-in-JS / CSS modules

If the plugin already uses a CSS approach, follow that pattern. Not recommended unless already established.

## `position: fixed` Caveat

The prototype uses `position: fixed` for header, config panel, and tooltip. Inside an Obsidian leaf panel, `position: fixed` positions relative to the viewport, not the leaf. This may be fine for main-pane use but breaks in sidebars.

**Investigate at runtime.** If issues appear, switch to `position: absolute` on the `.vault-heatmap-view` container (which must then have `position: relative; overflow: hidden`).

## Font

**Decision:** Use `var(--font-monospace)` — Obsidian's built-in monospace font. No external dependency. Replace all `'IBM Plex Mono',monospace` with `var(--font-monospace)` in the CSS.
