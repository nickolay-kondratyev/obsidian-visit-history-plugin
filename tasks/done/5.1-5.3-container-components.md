## 5.2 — TreemapViz.tsx (MOST COMPLEX)

**Props:**
```typescript
{
  data: VaultNode;  // the vault tree
  colorMode: 'type' | 'heatmap';
  gradKey: string;
  field: string;
  hotDays: number;
  coldDays: number;
  scales: Record<string, number>;
  onStatsChange: (stats: { files: number; folders: number; size: string }) => void;
}
```

### Hooks & State
- `containerRef` — for ResizeObserver
- `dims` — `{ w, h }` from container
- `hoveredIdx` — which leaf is hovered
- `tooltip` — `{ x, y, node }` or `null`

### D3 Layout (useMemo)
- `d3.hierarchy(data).sum(...)` + `d3.treemap()` — pure math, no DOM.
- Returns `{ folders, leaves }` arrays.

### Zoom (@visx/zoom — replaces d3-zoom)
This is the key architectural improvement over the prototype:

```tsx
import { Zoom } from '@visx/zoom';

// In render:
<Zoom
  width={dims.w}
  height={dims.h}
  scaleXMin={0.05} scaleXMax={30}
  scaleYMin={0.05} scaleYMax={30}
>
  {zoom => (
    <svg
      ref={svgRef}
      onMouseDown={zoom.dragStart}
      onMouseMove={zoom.dragMove}
      onMouseUp={zoom.dragEnd}
      onWheel={zoom.handleWheel}
      onDoubleClick={() => zoom.reset()}
    >
      <g transform={zoom.toString()}>
        {folders.map(d => <FolderNode key={...} d={d} />)}
        {leaves.map((d, i) => <LeafNode key={...} ... />)}
      </g>
    </svg>
  )}
</Zoom>
```

The `transform` is React-owned — no ref hacking. The reset-zoom button calls `zoom.reset()`.

### Notes
- Remove `svgRef`, `gRef`, `zoomRef` (no longer needed).
- Remove the zoom `useEffect` entirely.
- ResizeObserver and hover/tooltip logic ports directly from prototype.

## 5.3 — App.tsx

**Props:**
```typescript
{
  data: VaultNode;
}
```

### State (all owned here)
- `colorMode` — `'type' | 'heatmap'`
- `gradKey` — gradient preset key
- `field` — timestamp field
- `hotDays`, `coldDays` — heatmap thresholds
- `scales` — per-type size multipliers
- `configOpen` — config panel visibility
- `stats` — computed by TreemapViz, displayed by Header

### Render tree
```
<>
  <Header ... stats={stats} onConfigToggle={...} />
  <ConfigPanel ... />
  <TreemapViz data={data} ... onStatsChange={setStats} />
</>
```

### Key architectural note
App is **Obsidian-agnostic** — it receives `data` as a prop. The Obsidian integration lives entirely in `VaultTreemapView.ts` (Task 6.1). This keeps the component tree independently testable.
