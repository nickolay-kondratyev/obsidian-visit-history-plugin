# Task 8.1 & 8.2: Wire Into Plugin (main.ts Registration)

> **Phase:** 8 — Plugin Integration
> **Depends on:** Task 6.1
> **Blocks:** End-to-end testing

## 8.1 — Register View + Command in main.ts

Add to `VisitHistoryPlugin.onload()`:

```typescript
import { VaultTreemapView, VIEW_TYPE_TREEMAP } from './view/VaultTreemapView';

// Register the custom view type
this.registerView(
  VIEW_TYPE_TREEMAP,
  (leaf) => new VaultTreemapView(leaf, this),
);

// Add command to open the treemap
this.addCommand({
  id: 'open-vault-heatmap',
  name: 'Open vault heatmap',
  callback: () => {
    this.app.workspace.getLeaf(true).setViewState({
      type: VIEW_TYPE_TREEMAP,
      active: true,
    });
  },
});
```

## 8.4 — lastVisitedAt Wiring

The plugin already has `VisitHistoryService` and `FocusTracker` that track file visits. To feed `lastVisitedAt` into the treemap:

1. The `VisitHistoryService` (or settings) must expose visited timestamps as `Record<string, number>`.
2. Pass them to `buildVaultTree()` in `VaultTreemapView.refresh()`.

If the existing service doesn't expose this yet, ship with empty `visitedTimestamps` map — all files show "no data" for the visited field until tracking is hooked up. This is a **follow-up task**.
